How we measured this

Real agent runs, captured April 2026. Not estimates.

Two Claude agents were given the identical task: "create and pay a $10 USDC invoice on FlowLink." One agent used Playwright browser automation (MCP tools, DOM snapshots, click-based navigation). The other used FlowLink's markdown skill file -- a single curl fetch returning a 9 KB structured spec at /agent.md.

Token costs computed at Claude Sonnet 4.6 pricing: $3/M input, $15/M output. The Playwright agent was blocked at 25% completion and required an estimated 4 retries to match the markdown agent's outcome -- driving the effective cost ratio to 24x.

This gap is structural, not closeable

Browser automation cannot converge on markdown's efficiency. The overhead is architectural.

Why browser automation is expensive

  • A DOM snapshot is a 5-10 KB JSON tree per page, full of meta/script noise the LLM must parse
  • Each navigation requires hydration time (~2 sec) plus a fresh snapshot
  • Agents probe speculative URLs -- 404s and 401s burn tokens without progress
  • MCP transport wraps every call in JSON-RPC framing, adding overhead on every round-trip

Why markdown agents are cheap

  • One curl call returns 9 KB of dense, structured spec -- no noise
  • Markdown is already the LLM's native format -- zero parsing required
  • Skill files include error catalogues with agent_action hints, reducing retries
  • One file = no navigation latency, no hydration, no DOM noise, no speculative requests
